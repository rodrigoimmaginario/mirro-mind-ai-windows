# v0.14.0 — Conversation Intelligence

## Digest

This release closes CV13.E4, the Conversation Intelligence slice of the Mirror Web Experience. Conversations are now readable as transcript pages, Workspace conversation cards navigate directly to those transcripts, titles can be corrected manually, and a single conversation can request an explicit OpenRouter-backed title suggestion that is previewed before saving. The release also improves configuration awareness with detailed reference links and adds read-only journey attachment visibility.

## Highlights

- Added read-only conversation transcript pages.
- Linked Workspace conversation cards to transcript pages with mouse and keyboard navigation.
- Added manual conversation title editing through a bounded service endpoint.
- Added explicit single-conversation title suggestions through OpenRouter, with preview-only behavior and manual save approval.
- Added detailed Configuration Reference documentation and compact reference icons in the Configuration page.
- Added required/optional indicators for surfaced configuration items.
- Removed transcript export directory visibility from the Configuration surface because it is not yet a clear product-facing operation.
- Added read-only journey attachment visibility in Workspace.
- Refined Workspace journey sidebar and metric card density.

## Narrative context

`v0.13.0` made Mirror/runtime configuration visible and safer to inspect. `v0.14.0` turns conversation history into a usable local reading surface. Conversation cards are no longer dead summaries: they open transcripts, show speaker labels aligned with the active Mirror profile, and allow intentional title improvement.

The release keeps generation controlled. No LLM calls run on page load. Title suggestions happen only after clicking `Suggest title`, are not saved automatically, and must be approved through the existing manual `Save title` path.

## What changed

- Added `GET /api/conversations/detail` for bounded read-only transcript payloads.
- Added direct `#conversation/<id>` navigation.
- Added clickable and keyboard-accessible Workspace conversation cards.
- Added `ConversationService.update_title()` and `POST /api/conversations/title`.
- Added conversation title textbox to transcript pages.
- Added `ConversationService.suggest_title()` and `POST /api/conversations/title-suggestion`.
- Added `CONVERSATION_TITLE_PROMPT` and `generate_conversation_title()`.
- Added Configuration Reference docs under `docs/reference/configuration.md`.
- Added `docHref` and required/optional metadata to surfaced Configuration items.
- Added read-only Workspace Attachments tab and attachment metric for selected journeys.
- Propagated configured journey icons into the Workspace journey sidebar.

## Conscious exclusions

- No batch retitle execution.
- No LLM call on transcript page load.
- No automatic save after title suggestion.
- No message editing.
- No conversation deletion.
- No attachment upload, editing, or deletion.
- No raw configuration editor.
- No transcript export operation surfaced in the web UI.
